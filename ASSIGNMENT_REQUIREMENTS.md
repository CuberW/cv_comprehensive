## 三、系统设计要求

### 3.1 可理解性优先（核心考核点）

- **零黑盒**：每一个中间张量/矩阵都可以作为图像或热力图查看
- **参数游乐场**：每个参数都应该是一个可拖动的滑块，调整后即时重新渲染
- **叙事流**：每个算法以故事形式呈现——「这是输入 → 我们先做什么 → 为什么 → 这是结果」
- **直觉优先**：先展示效果建立直觉，数学细节默认折叠，按需展开
- **术语通俗化**：专业术语悬停即显示通俗解释

### 3.2 交互式体验

- 用户可上传自己的图片进行实验
- 提供精选示例图片，确保零等待体验
- 支持 before/after 对比滑块
- 支持算法参数实时调整

### 3.3 界面设计

- 清晰的信息架构，让用户知道「从哪里开始、接下来看什么」
- 算法按难度或类别分类
- 支持全局搜索

## 七、注意事项

1. 核心算法的代码不能由大模型生成，需要手工实现（可以参考课程 notebook 中的算法流程和公式）
2. 可以调用 OpenCV、NumPy 等基础库的矩阵运算和图像读写功能，但不能直接调用其高层 CV 算法函数（如 `cv2.Canny()`、`cv2.SIFT_create()` 等）
3. 前端 UI 可以使用任何框架（Flask + 原生 HTML/CSS/JS，或 Next.js / Vue 等），以最终用户体验为准
4. 视频需要真人出镜讲解，展示实际操作流程

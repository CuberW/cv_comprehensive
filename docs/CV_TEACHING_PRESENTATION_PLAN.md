# 计算机视觉通识教育系统正式页面规划

本文档用于指导后续开发：先把正式页面全部做扎实，最后再整理整体架构。

核心目标不是做一个“算法名词大全”，而是做一个让零基础观众能顺着页面操作、看图、看动画、看中间结果，逐步理解计算机视觉的通识教育系统。仓库里可以继续保留旧页面、兼容接口和实验模块，但最终首页、视频讲解和评分展示只围绕本文档中的正式范围展开。

## 1. 当前情况

当前项目已经有三类内容混在一起：

1. 课程作业积累下来的传统 CV 页面和算法。
2. 已经重构过或正在重构的重点教学页，例如色彩空间、直方图、阈值化、平滑去噪、Sobel、AI之眼。
3. 后来补进来的大量前沿模型或兼容模块，例如 Swin、DINO、MAE、ControlNet、DiT、FLUX、SAM2、DUSt3R、ORB-SLAM3 等。

问题在于：如果把所有名字都做成正式页面，每个页面都要有真实后端、步骤图、中间结果、动画、公式和讲解，工作量会非常大，而且最终容易变成“每页只有几段文字”的浅展示。

因此后续策略是：

- 正式展示范围收紧，只保留最能讲清 CV 知识链的内容。
- 旧页面和旧接口不强行删除，避免破坏已有实现；但不再作为首页重点入口。
- 深度学习和前沿模型只保留代表性算法，不再把所有前沿论文都纳入正式教学主线。
- 每个正式页面必须有后端真实计算或真实模型推理，不能只堆文字。

## 2. 明确不再作为正式页面的内容

以下内容不再作为正式页面或独立算法纳入后续主线：

- 独立非极大值抑制页。
- 模板匹配。
- Hough 变换。
- 轮廓查找。
- Swin、DINO、MAE、Grounding DINO、Mask2Former、BLIP-2。
- Stable Diffusion、StyleGAN、ControlNet、DiT、FLUX。
- SAM2、3D Gaussian Splatting、DUSt3R、ORB-SLAM3、ByteTrack、姿态估计大模型。

保留说明：

- Canny 内部的“非极大值抑制”仍然保留，因为它是 Canny 的必要步骤，但不单独做页面。
- Faster R-CNN / YOLO 内部的重叠框后处理可以展示为检测后处理，但不单独做 NMS 页面。
- “SIFT 特征匹配 + RANSAC”保留，因为它是局部特征与几何一致性，不是模板匹配。
- 被排除的前沿模型可以在最后的“未来扩展”里用一两句话提到，但不承诺完整交互页面。

## 3. 正式课程主线

最终系统建议按以下路线组织：

```text
图像是什么
  -> 色彩、灰度、直方图、阈值、噪声
  -> 平滑、卷积、Sobel、Canny
  -> 角点、特征、匹配、传统分割、几何与运动
  -> CNN 如何学习视觉特征
  -> AI之眼：检测、语义分割、实例分割
  -> 生成模型：GAN 与扩散
  -> Transformer 与基础模型：ViT、DETR、CLIP、SAM、NeRF
```

正式首页建议保留纵向专题卡，不再展示零散单算法卡。每个专题内部可以有多个算法标签页或步骤段落。

| 专题 | 正式页面 | 包含内容 | 定位 |
|---|---|---|---|
| 图像基础 | `colorspace.html`、`grayscale.html`、`histogram_new.html`、`threshold_new.html`、`noise.html` | 色彩、亮度、统计、二值化、噪声 | 让观众理解图像是矩阵 |
| 滤波与边缘 | `smoothing.html`、卷积页、`sobel.html`、`edge.html` | 平滑去噪、卷积核、梯度、边缘检测 | 从像素变化进入结构 |
| 传统 CV | 角点、SIFT、匹配、传统分割、几何与运动页面 | 人工设计特征和几何规则 | 连接传统视觉和深度学习 |
| AI之眼 | `detection_segmentation.html` | Faster R-CNN、YOLO、FCN、U-Net、Mask R-CNN | 项目重点页 |
| 深度基石 | CNN 基础专题页 | CNN、LeNet/训练过程、ResNet、Grad-CAM | 解释深度视觉为什么有效 |
| 生成模型 | 生成模型专题页 | GAN、扩散模型 | 解释 AI 如何生成图像 |
| Transformer 与基础模型 | 前沿基础模型专题页 | ViT、DETR、CLIP、SAM、NeRF | 解释现代 CV 新范式 |

## 4. 每个正式页面的统一标准

每个正式页面必须包含：

| 区块 | 要求 |
|---|---|
| 一句话问题 | 这个算法解决什么问题 |
| 输入 | 原图、构造数据或用户上传图 |
| 步骤 | 至少 4 个关键步骤 |
| 中间结果 | 每步有真实图像、图表、矩阵、mask、框或 token 数据 |
| 公式 | 给出核心公式，不堆推导 |
| 通俗解释 | 说明这一步具体在做什么 |
| 动画/示意 | 用滑窗、排序、注意力、射线、mask 生长等动画解释过程 |
| 最终结果 | 与输入或其他算法对比 |
| 应用场景 | 告诉观众真实世界中什么时候用 |
| 局限性 | 告诉观众它会失败在什么地方 |

后端响应建议统一为：

```json
{
  "module_id": "algorithm_id",
  "requested_module_id": "frontend_id",
  "steps": [
    {
      "id": "input",
      "name": "输入图像",
      "image_base64": "...",
      "explanation": "这一步具体做了什么",
      "formula": "I(x,y)",
      "data": {}
    }
  ],
  "metrics": {
    "problem": "解决的问题",
    "applications": [],
    "limitations": []
  },
  "implementation": {
    "category": "numpy_algorithm | local_mechanism | pretrained_model",
    "real_model": false
  }
}
```

实现标记必须诚实：

- `pretrained_model / real_model:true`：确实加载并运行预训练模型。
- `numpy_algorithm / real_model:false`：传统算法或本地 NumPy/Pillow 实现。
- `local_mechanism / real_model:false`：小型机制演示，结构正确，但不是大型预训练模型推理。

## 5. 图像基础页面规划

### 5.1 色彩空间

正式页面：`colorspace.html`

整体意义：同一张图片可以在不同颜色坐标系中表达。RGB 适合屏幕显示，HSV 适合按颜色选择，Lab 适合感知色差，CMYK 适合印刷分色。

后端步骤：

1. 输入图像：返回原图、尺寸、通道数。
2. RGB 拆分：返回 R/G/B 三个保留单通道颜色的图。
3. HSV 拆分：返回 H 色相图、S 饱和度灰度图、V 明度灰度图。
4. Lab 拆分：返回 L 亮度图、a 绿红发散图、b 蓝黄发散图。
5. CMYK 拆分：返回 C/M/Y/K 四个油墨覆盖图，纯黑时避免除零。

前端展示：

- 顶部四个概念图：RGB 三圆相交、HSV 色相环、Lab 的 L 轴与 a/b 平面、CMYK 油墨叠印。
- 上传同一张图后纵向展示四种模式的通道拆分。
- 每个通道卡显示含义、取值范围、公式、应用场景。

动画/示意：

- RGB 三圆相交发光动画。
- HSV 色相环旋转，S/V 用半径和亮度解释。
- Lab 用 a/b 平面上的点移动解释颜色差。
- CMYK 用油墨叠印层逐层覆盖。

### 5.2 灰度化

正式页面：`grayscale.html`

整体意义：把三通道颜色压缩成一个亮度通道，是很多传统算法的第一步。

后端步骤：

1. 输入 RGB。
2. 三通道拆分。
3. 加权亮度计算。
4. 灰度图输出。
5. 原图/灰度图差异说明。

公式：

```text
Y = 0.299R + 0.587G + 0.114B
```

前端展示：

- 显示三个通道权重条。
- 鼠标移动到某个像素时显示 RGB 和灰度值。
- 说明为什么绿色权重最高：人眼对绿色更敏感。

### 5.3 直方图均衡化

正式页面：`histogram_new.html`

整体意义：把集中在狭窄亮度范围的像素重新拉开，提高对比度。

后端步骤：

1. 输入图像。
2. 灰度化。
3. 统计原始直方图。
4. 计算累计分布 CDF。
5. 构造灰度映射函数。
6. 按映射重新分配像素。
7. 输出均衡化结果和新直方图。

公式：

```text
T(k)=round((CDF(k)-CDF_min)/(N-CDF_min)*255)
```

前端展示：

- 原图和结果图对比滑块。
- 原始直方图与均衡后直方图动态条形图。
- CDF 曲线逐步累积动画。
- 像素亮度沿映射函数移动的动画。

应用：

- 低光照照片增强。
- 医学和工业图像对比度增强。
- 扫描文档改善。
- 遥感图像增强。

局限：

- 可能放大噪声。
- 对本来对比度已经很强的图像可能过度增强。

### 5.4 阈值化

正式页面：`threshold_new.html`

整体意义：把连续灰度图变成前景/背景二值图，是从“看亮度”到“分区域”的第一步。

后端步骤：

1. 输入图像。
2. 灰度化。
3. 绘制灰度直方图。
4. 固定阈值分割。
5. Otsu 自动阈值计算。
6. 输出二值图和前景比例。

公式：

```text
B(x,y)=1 if I(x,y) >= T else 0
```

Otsu 展示：

```text
T* = argmax sigma_b^2(T)
```

前端展示：

- 阈值滑块实时移动，直方图上显示竖线。
- 图像像素随阈值变化从灰度变成黑白。
- Otsu 用类间方差曲线解释“最佳分割点”。

应用：

- 文档扫描。
- 工业检测。
- 简单前景提取。
- 医学二值 mask 初筛。

### 5.5 噪声模型

正式页面：`noise.html`

整体意义：现实图像不干净，理解噪声才能理解为什么要滤波。

后端步骤：

1. 输入干净图。
2. 添加高斯噪声。
3. 添加椒盐噪声。
4. 添加泊松/散粒噪声。
5. 添加斑点噪声。
6. 输出噪声统计和局部放大图。

前端展示：

- 一张图变成多个噪声版本。
- 每种噪声有局部放大镜。
- 显示噪声分布曲线。
- 告诉观众对应推荐滤波：高斯噪声适合高斯/双边，椒盐噪声适合中值。

## 6. 滤波、卷积与边缘页面规划

### 6.1 平滑与去噪

正式页面：`smoothing.html`

包含算法：高斯平滑、中值滤波、双边滤波。

整体意义：三种算法都在去噪，但面对的噪声和保边能力不同。

后端统一步骤：

1. 输入图像。
2. 构造同一组噪声图。
3. 高斯平滑管线。
4. 中值滤波管线。
5. 双边滤波管线。
6. 三算法结果对比。
7. 噪声降低、边缘保留、细节损失指标。

高斯平滑步骤：

- 构造高斯核。
- 滑动窗口取局部区域。
- 核权重与像素相乘。
- 加权求和得到中心像素。
- 输出平滑结果和细节损失图。

中值滤波步骤：

- 取 3x3 或 5x5 窗口。
- 展开窗口像素。
- 排序。
- 选择中间值。
- 替换中心像素。

双边滤波步骤：

- 计算空间距离权重。
- 计算颜色/亮度相似权重。
- 两种权重相乘。
- 归一化加权求和。
- 输出保边去噪结果。

前端展示：

- 严格搬入 Canny 页面里的高斯平滑动画，并扩展讲解核权重。
- 中值滤波做“窗口像素飞入排序条，中位数高亮”的动画。
- 双边滤波做“空间近但颜色差的像素被降权”的动画。
- 页面底部做三算法对比表：适用噪声、优点、缺点、速度、保边能力。

### 6.2 卷积

正式页面：卷积专题页，或 `teaching.html?id=convolution`

整体意义：卷积是传统滤波和 CNN 的共同语言。

后端步骤：

1. 输入图像。
2. 选择卷积核。
3. 显示局部窗口。
4. 窗口与核逐元素相乘。
5. 求和生成输出像素。
6. 输出完整卷积结果。

前端展示：

- 核矩阵和图像窗口同步移动。
- 每个格子显示乘法贡献。
- 切换平滑核、锐化核、边缘核。
- 连接到 CNN：传统卷积核人工设计，CNN 卷积核由数据学习。

### 6.3 Sobel

正式页面：`sobel.html`

整体意义：Sobel 用两个方向的卷积核计算亮度变化，找出边缘方向和强度。

后端步骤：

1. 输入灰度图。
2. Sobel X 卷积。
3. Sobel Y 卷积。
4. 梯度幅值计算。
5. 梯度方向计算。
6. 输出边缘强度图和方向图。

公式：

```text
G = sqrt(Gx^2 + Gy^2)
theta = atan2(Gy, Gx)
```

前端展示：

- 严格搬入 Canny 页面里的 Sobel X/Y 动画。
- 新增幅值合成动画：X 和 Y 两个箭头合成一个梯度箭头。
- 显示不同方向边缘为什么分别在 Gx/Gy 中更明显。

### 6.4 Canny

正式页面：`edge.html`

整体意义：Canny 把去噪、梯度、细化、双阈值连接组合成稳定边缘检测流程。

后端步骤：

1. 输入图像。
2. 灰度化。
3. 高斯平滑。
4. Sobel 梯度。
5. 非极大值抑制。
6. 双阈值分类强边/弱边。
7. 滞后连接。
8. 最终边缘图。

前端展示：

- 保留已有高质量动画。
- 每个阶段给一张真实中间图。
- 非极大值抑制只作为 Canny 内部步骤，不作为独立页面。

## 7. 传统 CV 页面规划

这一组是“人工设计规则”的代表，建议保留为正式二级专题，不必在首页拆成太多单卡。

### 7.1 角点检测

包含：Harris、Shi-Tomasi。

整体意义：角点是两个方向变化都强的位置，适合跟踪、匹配和几何估计。

步骤：

1. 输入灰度图。
2. 计算 Ix/Iy 梯度。
3. 构造二阶矩矩阵。
4. 计算 Harris 响应或最小特征值。
5. 阈值筛选。
6. 局部极大值筛选。
7. 输出角点叠加图。

动画：

- 小窗口在平坦区、边缘区、角点区移动。
- 显示窗口向不同方向移动时像素变化强弱。

### 7.2 SIFT

整体意义：SIFT 在尺度和旋转变化下仍能找到稳定关键点和描述子。

步骤：

1. 构建尺度空间。
2. 计算 DoG。
3. 3D 极值检测。
4. 关键点定位。
5. 主方向分配。
6. 4x4x8 描述子生成。
7. 输出关键点和描述子摘要。

动画：

- 图像金字塔层级切换。
- DoG 极值点闪烁。
- 方向直方图旋转对齐。
- 128 维描述子条形图生成。

### 7.3 SIFT 特征匹配 + RANSAC

整体意义：用局部描述子找到两张图中的对应点，再用几何模型剔除错配。

注意：这是特征匹配，不是模板匹配。

步骤：

1. 两张图分别提取 SIFT 关键点。
2. 计算描述子距离。
3. 最近邻匹配。
4. ratio test 过滤。
5. RANSAC 随机采样。
6. 估计单应矩阵或基础矩阵。
7. 输出内点匹配图。

动画：

- 匹配线先很多很乱。
- ratio test 删除明显错配。
- RANSAC 多轮抽样，最后只保留几何一致的匹配。

### 7.4 传统分割

建议做成一个“传统分割”专题页，而不是首页分散成多个卡。

包含：K-Means、Watershed、GrabCut、SLIC、Normalized Cuts。

展示方式：

- 同一张图，切换不同传统分割算法。
- 每个算法给 4 到 6 个步骤。
- 页面底部对比：是否需要用户交互、是否依赖颜色、是否能处理粘连目标、边界是否细。

K-Means 步骤：

1. 像素转成颜色/位置特征。
2. 随机或均匀初始化聚类中心。
3. 分配像素到最近中心。
4. 更新中心。
5. 迭代收敛。
6. 输出颜色聚类分割图。

Watershed 步骤：

1. 灰度或梯度图。
2. 生成前景/背景 marker。
3. 模拟水从 marker 淹没。
4. 相遇处形成分水岭。
5. 输出区域边界。

GrabCut 步骤：

1. 用户给初始框。
2. 初始化前景/背景。
3. 建立颜色模型。
4. 图割优化。
5. 迭代更新 mask。
6. 输出抠图结果。

SLIC 步骤：

1. 初始化规则网格中心。
2. 在局部区域内按颜色和空间距离聚类。
3. 更新超像素中心。
4. 输出超像素边界。

NCuts 步骤：

1. 构建像素/区域图。
2. 计算相似度矩阵。
3. 求解谱向量。
4. 根据向量切分区域。
5. 输出分割结果。

### 7.5 几何与运动

建议做成“几何与运动”专题页，内部包含相机标定、极线几何、SfM、立体匹配、光流、频域分析。

目的不是做大而全，而是讲清“从二维图像恢复空间关系和运动”的思想。

相机标定步骤：

1. 棋盘格角点。
2. 世界坐标与图像坐标对应。
3. 求内参和畸变。
4. 重投影误差。
5. 畸变校正前后对比。

极线几何步骤：

1. 两视图匹配点。
2. 估计基础矩阵。
3. 点生成对应极线。
4. 点到线距离验证。
5. 极线可视化。

SfM 步骤：

1. 多视角匹配。
2. 估计相机相对位姿。
3. 三角化稀疏 3D 点。
4. 重投影验证。
5. 输出相机轨迹和点云预览。

立体匹配步骤：

1. 左右图校正。
2. 同一行搜索匹配。
3. 计算视差。
4. 视差转深度。
5. 输出深度图。

光流步骤：

1. 连续两帧。
2. 亮度恒定假设。
3. 局部窗口求运动。
4. 输出流场箭头。
5. 颜色编码运动方向。

频域分析步骤：

1. 输入灰度图。
2. 计算 2D FFT。
3. 显示频谱。
4. 低通/高通滤波。
5. 逆变换结果。

## 8. AI之眼页面规划

正式页面：`detection_segmentation.html`

这是项目最重要的页面。目标是让观众用同一张图理解三种视觉任务：

- 目标检测：图里有什么，它们在哪里，用框表示。
- 语义分割：每个像素属于什么类别，同类不区分个体。
- 实例分割：每个像素属于哪个独立物体，同类也要分开。

页面包含五个算法标签：Faster R-CNN、YOLO、FCN、U-Net、Mask R-CNN。

### 8.1 后端策略

真实预训练模型优先：

- Faster R-CNN：使用 torchvision 预训练权重。
- FCN / DeepLabV3：用于语义分割真实结果，FCN 作为主讲。
- Mask R-CNN：使用 torchvision 预训练权重。

需要特殊处理：

- YOLO：torchvision 不提供官方 YOLO。建议第一阶段使用可选小权重 YOLOv8n/YOLO11n，通过外部缓存下载，不放入仓库；如果环境没有安装对应依赖，页面要明确提示下载方式，不返回假 YOLO 结果。
- U-Net：通用预训练 U-Net 不如 torchvision 分割模型稳定。建议做“小型 U-Net 机制演示 + 可选训练脚本”，用合成形状或小数据集展示跳跃连接和边界恢复。真实语义结果仍由 FCN/DeepLabV3 支撑。

### 8.2 Faster R-CNN 展示

整体意义：两阶段检测。第一阶段找可能有目标的区域，第二阶段精细分类和修正框。

步骤：

1. 输入图像。
2. 预处理成张量。
3. Backbone/FPN 特征摘要。
4. RPN 生成候选区域。
5. RoI Align 对候选区域取特征。
6. 分类分支输出类别分数。
7. 框回归分支修正位置。
8. 重叠框后处理和置信度过滤。
9. 最终检测框叠加图。

中间结果：

- 候选框热力图。
- Top-K 候选框。
- 类别分数表。
- 最终框、类别、置信度。

动画：

- 大量候选框出现。
- 低分框逐渐消失。
- 剩余框被修正到物体边界。

### 8.3 YOLO 展示

整体意义：单阶段检测。整张图只看一次，直接输出目标框、类别和置信度。

步骤：

1. 输入图像。
2. 图像划分网格。
3. 每个网格预测框。
4. 预测 objectness。
5. 预测类别概率。
6. 重叠框后处理。
7. 最终检测结果。

中间结果：

- 网格图。
- objectness 热力图。
- 候选框集合。
- 最终框叠加图。

动画：

- 每个网格同时发出预测框。
- 置信度低的框淡出。
- 高分框保留。

实现说明：

- 有真实 YOLO 小权重时展示真实推理结果。
- 没有权重时不伪造结果，只展示机制演示并给出下载说明。

### 8.4 FCN 展示

整体意义：把分类网络改成全卷积网络，让每个像素都得到类别预测。

步骤：

1. 输入图像。
2. Backbone 提取低分辨率特征。
3. 生成类别 logits。
4. 上采样回原图尺寸。
5. 像素 softmax。
6. argmax 得到类别标签图。
7. overlay 到原图。

中间结果：

- 类别概率摘要。
- 置信度热力图。
- 标签图。
- 类别面积统计。

动画：

- 粗分割图逐步放大回原图。
- 每个像素从概率分布变成最终类别颜色。

### 8.5 U-Net 展示

整体意义：编码器压缩语义，解码器恢复空间分辨率，跳跃连接把细节带回来。

步骤：

1. 输入图像或合成数据。
2. 编码器逐层下采样。
3. bottleneck 获取高级语义。
4. 解码器逐层上采样。
5. 跳跃连接拼接同尺度特征。
6. 输出像素 mask。
7. 显示边界恢复效果。

中间结果：

- 每层 feature map。
- 下采样尺寸变化。
- 跳跃连接箭头。
- 无跳跃/有跳跃边界对比。

动画：

- 左侧编码器向下压缩。
- 右侧解码器向上恢复。
- 横向跳跃连接把纹理细节送回去。

实现说明：

- 默认做本地小型 U-Net 机制实现，`real_model:false`。
- 可选提供训练脚本，训练数据和权重不强制放仓库。

### 8.6 Mask R-CNN 展示

整体意义：在 Faster R-CNN 的框基础上，为每个实例单独预测 mask。

步骤：

1. 输入图像。
2. 检测框预测。
3. RoI Align 提取每个实例区域。
4. mask 分支输出 mask 概率。
5. 阈值化得到二值 mask。
6. 每个实例独立着色。
7. 输出实例分割 overlay。

中间结果：

- 检测框。
- 每个实例的 mask 概率图。
- 二值 mask。
- 实例面积、类别、置信度表。

动画：

- 框先定位物体。
- 每个框内部逐渐长出 mask。
- 同类物体被分成不同颜色。

## 9. 深度基石页面规划

正式页面：建议做一个统一专题页，例如 `deep_foundations.html` 或现有深度学习入口。

包含：CNN 基础、LeNet/卷积训练过程、ResNet、Grad-CAM。

### 9.1 CNN 基础

整体意义：CNN 用卷积层提局部特征，用 ReLU 增加非线性，用池化压缩信息，用全连接或分类头输出类别。

步骤：

1. 输入图像。
2. 卷积层提取局部模式。
3. ReLU 去掉负响应。
4. Pool 降采样。
5. 多层堆叠形成高级特征。
6. 分类头输出概率。

中间结果：

- 卷积核。
- feature map。
- ReLU 前后对比。
- Pool 前后尺寸变化。
- 分类概率条形图。

动画：

- 卷积核滑动。
- ReLU 把负值压成 0。
- Pool 窗口取最大值。

### 9.2 LeNet / 卷积训练过程

整体意义：用一个小网络展示神经网络不是魔法，而是通过损失函数和反向传播逐步调整权重。

步骤：

1. 输入小图像或 MNIST 示例。
2. 前向传播。
3. 得到预测概率。
4. 计算交叉熵损失。
5. 反向传播得到梯度。
6. 更新权重。
7. 损失曲线下降。

中间结果：

- 每层张量尺寸。
- 卷积核权重。
- 预测概率。
- loss 曲线。
- 几个 epoch 的错误样本变化。

实现建议：

- 可手写一个轻量 PyTorch 训练脚本。
- 使用小数据集或内置样例，不把大型数据集放仓库。
- 这部分最适合体现“核心代码自己实现”。

### 9.3 ResNet

整体意义：残差连接让网络学习“修正量”而不是完整映射，缓解深层网络退化。

步骤：

1. 输入特征。
2. 普通卷积分支计算 F(x)。
3. shortcut 保留 x。
4. 两路相加得到 F(x)+x。
5. 多个残差块堆叠。
6. 输出分类结果。

公式：

```text
y = F(x) + x
```

中间结果：

- 残差块结构图。
- 普通网络和残差网络梯度流对比。
- 真实预训练 ResNet 分类 Top-K。

动画：

- 主分支和 shortcut 两条路径汇合相加。
- 深层网络中梯度沿 shortcut 回流。

### 9.4 Grad-CAM

整体意义：解释分类网络“看哪里”，用梯度加权最后一层特征图生成热力图。

步骤：

1. 输入图像。
2. ResNet 分类得到目标类别。
3. 取目标类别对 feature map 的梯度。
4. 梯度全局平均得到通道权重。
5. 加权求和生成 CAM。
6. ReLU 和上采样。
7. 热力图叠加原图。

公式：

```text
L^c = ReLU(sum_k alpha_k^c A^k)
```

中间结果：

- 分类概率。
- 目标层 feature map 摘要。
- CAM 热力图。
- overlay。

动画：

- 类别分数反向影响 feature map。
- 热区逐渐亮起。

## 10. 生成模型页面规划

正式页面：建议做一个统一专题页，例如 `generative_models.html`。

包含：GAN、扩散模型。

### 10.1 GAN

整体意义：生成器造假，判别器打假，两者对抗训练，生成器逐渐学会生成像真的样本。

步骤：

1. 随机噪声 z。
2. 生成器生成样本。
3. 判别器判断真/假。
4. 计算 D loss。
5. 计算 G loss。
6. 参数更新。
7. 展示生成结果随训练变化。

中间结果：

- 噪声向量。
- 生成样本。
- 判别器真假分数。
- G/D loss 曲线。
- 训练轮次下的结果变化。

动画：

- G 把噪声变成图。
- D 给出真假判断。
- 损失箭头回传给 G。

实现策略：

- 默认做 toy GAN 或小图像机制演示。
- 不追求大型高清生成，不放大权重。
- 标记 `local_mechanism / real_model:false`，除非未来接入真实权重。

### 10.2 扩散模型

整体意义：先学习把图像逐步加噪，再学习从噪声一步步还原图像。

步骤：

1. 输入图像。
2. 前向加噪时间轴。
3. 不同时间步 noisy image。
4. 噪声预测网络机制。
5. 单步去噪。
6. 多步反向去噪。
7. 输出恢复结果。

公式：

```text
x_t = sqrt(alpha_bar_t) x_0 + sqrt(1-alpha_bar_t) epsilon
```

中间结果：

- t=0、t=25、t=50、t=100 的加噪图。
- 预测噪声图。
- 去噪轨迹。
- 结果对比。

动画：

- 从清晰图逐步变成噪声。
- 再从噪声逐步变回结构。

实现策略：

- 默认做小型本地机制演示。
- 不把 Stable Diffusion 作为正式页面。

## 11. Transformer 与基础模型页面规划

正式页面建议分成两个专题：

1. Transformer 统一视觉：ViT、DETR。
2. 基础模型时代：CLIP、SAM、NeRF。

### 11.1 ViT

整体意义：把图像切成 patch，把 patch 当成 token，用 Transformer 做视觉理解。

步骤：

1. 输入图像。
2. 切成固定大小 patch。
3. patch 展平并线性映射成 token。
4. 加位置编码。
5. self-attention 计算 token 关系。
6. class token 聚合全局信息。
7. 分类输出。

中间结果：

- patch 网格。
- token 序列。
- attention map。
- 分类 Top-K。

动画：

- 图像变成一串 patch token。
- 选中一个 patch，显示它关注其他 patch 的强弱。

实现策略：

- 如果使用 torchvision ViT 预训练权重，则 `real_model:true`。
- 如果只做机制演示，则 `real_model:false`，不能冒充真实分类。

### 11.2 DETR

整体意义：把目标检测看成集合预测，用 object query 直接预测一组目标。

步骤：

1. 输入图像。
2. backbone 提取特征。
3. Transformer encoder 建立全局关系。
4. object queries 进入 decoder。
5. 每个 query 输出类别和框。
6. 匈牙利匹配把预测和真实目标一一对应。
7. 输出最终检测框。

中间结果：

- query attention。
- query 输出框。
- 匹配线示意。
- 最终框。

动画：

- 100 个 query 像问题一样询问图像。
- 有些 query 变成目标，有些变成 no-object。
- 匈牙利匹配把预测框和目标连接。

实现策略：

- 优先接入官方或 Hugging Face 预训练权重并走外部缓存。
- 没有权重时展示集合预测机制，不返回假检测结果。

### 11.3 CLIP

整体意义：把图像和文本映射到同一个向量空间，支持零样本识别。

步骤：

1. 输入图像。
2. 输入候选文本。
3. 图像编码器得到 image embedding。
4. 文本编码器得到 text embeddings。
5. 计算余弦相似度矩阵。
6. softmax 得到文本匹配概率。
7. 输出 Top-K 文本。

中间结果：

- 图像向量。
- 文本向量。
- 相似度热力表。
- Top-K 结果。

动画：

- 图像点和文本点在同一空间靠近。
- 错误文本离得远，正确文本离得近。

实现策略：

- 可使用 open_clip 或 transformers 预训练权重，权重放外部缓存。
- 没有权重时只展示机制，不伪造零样本结果。

### 11.4 SAM

整体意义：把分割变成提示驱动任务。用户给点或框，模型输出对应区域 mask。

步骤：

1. 输入图像。
2. 用户点/框提示。
3. image encoder 提取图像特征。
4. prompt encoder 编码提示。
5. mask decoder 融合图像和提示。
6. 输出多个候选 mask。
7. 根据 IoU 分数选择 mask。

中间结果：

- 用户提示。
- 候选 mask。
- IoU 分数。
- 最终 overlay。

动画：

- 点击一个点，附近物体区域被点亮。
- 拖一个框，框内目标生成 mask。

实现策略：

- 如果接入 SAM 权重，权重不放仓库，文档说明下载位置。
- 如果环境没有权重，返回清晰下载提示，不伪造 SAM 结果。

### 11.5 NeRF

整体意义：用一个神经网络隐式表示 3D 场景。沿相机射线采样，预测每个点的颜色和密度，再体渲染成图像。

步骤：

1. 输入相机位姿和像素射线。
2. 沿射线采样 3D 点。
3. 对点做位置编码。
4. MLP 预测颜色和密度。
5. 体渲染积分。
6. 输出新视角图像或射线颜色。

公式：

```text
C(r)=sum_i T_i (1-exp(-sigma_i delta_i)) c_i
```

中间结果：

- 相机射线图。
- 采样点。
- 密度分布。
- 颜色累积过程。
- 简单新视角预览。

动画：

- 一条射线穿过场景。
- 采样点按密度贡献颜色。
- 多条射线组成一张图。

实现策略：

- 做 toy NeRF 机制演示即可，不要求训练真实复杂场景。
- 标记 `local_mechanism / real_model:false`。

## 12. 模型权重、训练与原创性策略

### 12.1 权重管理

大权重不放仓库。推荐策略：

- 小型教学权重：如果体积很小，可以放入 `assets/models` 或 release。
- torchvision 官方权重：使用 torch cache 或 `CV_MODEL_DIR`。
- YOLO、CLIP、SAM、DETR 等外部权重：使用下载脚本或说明文档，默认放外部缓存。
- 文档中写清楚模型来源、license、大小、缓存路径和下载命令。

### 12.2 训练管理

第一版不自动训练大模型。

可选训练只用于：

- LeNet / CNN 教学训练。
- 小型 U-Net 合成数据训练。
- toy GAN。
- toy diffusion。
- toy NeRF。

训练脚本要能独立运行，训练结果不强制进入仓库。

### 12.3 原创性说明

保证书不能写成“所有代码和模型都从零原创”。更合理、更诚实的表述是：

- 传统算法核心计算由项目代码实现，不使用 OpenCV 直接调用成品算法。
- 深度学习大型模型使用公开预训练权重和公开框架，并在文档中明确标注来源。
- 本项目原创重点在于：教学系统组织、步骤拆解、中间结果生成、可视化交互、传统算法实现和小型机制演示。
- 使用外部模型时，不把模型权重和论文成果冒充为自己训练。

## 13. 后续开发顺序
